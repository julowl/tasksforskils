Теория (примерно 3 минуты чтения)

Зачем классы для матриц?
- Инкапсуляция: класс Matrix объединяет данные и поведение (валидация, арифметика, преобразования).
- Повторное использование: методы вроде transpose или multiplication естественно вызываются у экземпляров.
- Операторы: Python поддерживает специальные методы (__add__, __matmul__), делающие код более идиоматичным.

Представление матриц
- Простое внутреннее представление — вложенные списки: List[List[float]].
- Требуется проверка прямоугольности: все строки должны иметь одинаковую длину. Пустая матрица представляется как пустой список [].
- Используйте глубокие копии при возврате или сохранении данных, чтобы избежать непреднамеренных изменений извне.

Базовые операции
- shape: возвращает (rows, cols). Для [] shape = (0,0).
- is_square: True когда rows == cols и rows > 0.
- zeros и identity: классовые методы для создания матриц без явной передачи списков.

Индексация и изменение
- __getitem__ может быть гибким: m[i] возвращает i-ю строку, m[i,j] возвращает элемент.
- __setitem__ поддерживает присваивание элемента m[i,j] = value.
- row(r) и col(c) возвращают копии соответствующих строк/столбцов.

Арифметика
- Поэлементное сложение/вычитание требуют одинаковой формы; при несовпадении выбрасывайте ValueError.
- Умножение на скаляр должно возвращать новый объект Matrix; реализуйте __mul__ для удобства.
- Не изменяйте исходные операнды, если это не оговорено.

Умножение
- Матрица A умножается на B, если A.cols == B.rows.
- Умножение на вектор возвращает список float — результат умножения на столбец.
- Hadamard — поэлементное умножение, требует одинаковой формы.

Продвинутые операции
- transpose меняет строки и столбцы. Транспонированная пустая матрица — пустая.
- trace суммирует диагональные элементы — только для квадратных матриц.
- determinant можно реализовать рекурсивно через разложение по минорам (подходит для небольших матриц в упражнениях).

Сравнение и допуски
- Точное сравнение требует одинаковой формы и точных значений float.
- Приближённое сравнение учитывает допуски и полезно при вычислениях с плавающей точкой.

Как использовать упражнения
- В каждом файле методы реализованы как заглушки, которые бросают NotImplementedError.
- Реализуйте методы, чтобы пройти doctest и набор unit-тестов внизу файла.
- Запускайте python <file>.py для запуска тестов и поэтапной реализации.
